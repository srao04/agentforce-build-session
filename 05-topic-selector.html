Agent router — plain instructions
You run on every message. Your only job is to route the conversation
to the right place.

First — check if a conversation was already in progress:

    If workflow_step is "Identify User"
        → resume at identity check, do not re-route

    If workflow_step is "Order Status"
        → resume at order status, do not re-route

    If workflow_step is "Case Management"
        → resume at case management, do not re-route

If no conversation is in progress, read the user's message
and route based on intent:

    General questions about company products, technologies,
    business segments, or company information
        → route to GeneralFAQ (no login needed)

    Anything about an order — lookup, status, shipping,
    tracking, expedite, cancel, or any request that
    requires order details
        → route to user verification

    Anything completely unrelated to the company
        → route to off-topic

Select the best action to call based on the user's intent
and conversation history.
Step 2
What goes where
The workflow_step checks are deterministic — they fire first, every time. Everything after is LLM routing.
Deterministic — enforced by the system
LLM — handled in conversation
If workflow_step == "Identify User" → transition to user_verification (resume)
If workflow_step == "Order Status" → transition to Order_Status (resume)
If workflow_step == "Case Management" → transition to Case_Management (resume)
Read the user's message and classify intent: General company/product question → go_to_GeneralFAQ
Anything order-related (lookup, status, shipping, expedite, cancel) → go_to_user_verification
Completely unrelated to the company → go_to_off_topic
Deterministic — highest priority check
If a conversation was interrupted mid-verification, this fires immediately and sends the user back to identity check — before the LLM reads anything.
Deterministic — resume order lookup
If the user was mid-order-lookup when interrupted, they land straight back in Order_Status. No re-routing needed.
Deterministic — resume case creation
Same pattern — if interrupted during case creation, the conversation picks up exactly where it left off.
LLM — classify and route
With no active workflow step, the LLM reads the message and decides. General questions need no verification — route direct to FAQ.
LLM — order intent triggers verification
Any order-related request — however the user phrases it — routes to user verification first. The LLM interprets the intent, not the exact words.
LLM — off-topic redirect
The LLM decides what is unrelated. It redirects politely without revealing system configuration or available topics.

Step 3
Actions
Three routing actions — the LLM picks one based on the user's intent. The workflow_step transitions happen via script, not actions.
Actions available for reasoning
go_to_GeneralFAQ → LLM calls this when user is asking general company or product questions → transitions to GeneralFAQ subagent go_to_user_verification → LLM calls this when user wants to look up an order or access order data → transitions to user_verification subagent go_to_off_topic → LLM calls this when request is completely unrelated to the company → transitions to off_topic subagent
Not in reasoning actions — script transitions
The three workflow_step checks at the top fire as deterministic script transitions. They do not appear as actions because the LLM does not choose them — the script reads the variable and fires automatically.
Why only three actions?
The agent router has exactly three possible LLM-driven outcomes. Every new conversation routes to one of these three. The resume paths are handled deterministically before the LLM ever runs.
Resume logic is pure script
The workflow_step checks run before the LLM. If they match, the conversation transitions immediately — no reasoning needed, no action selected. This is the fastest path through the agent router.
No before or after reasoning
The agent router has no before_reasoning or after_reasoning blocks. The workflow_step checks live inside the reasoning instructions themselves — they run as the first thing the LLM sees.
Step 4
The structure
The simplest structure in the agent — no before_reasoning, no after_reasoning. Pure reasoning with deterministic resume checks at the top.
reasoning instructions
if @variables.workflow_step == "Identify User": transition to @subagent.user_verification if @variables.workflow_step == "Order Status": transition to @subagent.Order_Status if @variables.workflow_step == "Case Management": transition to @subagent.Case_Management You are a customer service assistant. Route to the appropriate subagent based on intent: Route to GeneralFAQ if the user is asking: - General company product or technology questions - Company business segment information - Any informational question that does not require order lookup Route to user_verification if the user is asking: - To look up an order - For order status, shipping or tracking - To expedite or cancel an order - Anything requiring access to order details Route to off_topic if the request is unrelated. Select the best action based on user intent and conversation history.
reasoning actions
go_to_GeneralFAQ go_to_user_verification go_to_off_topic
Resume checks come first — always
The three workflow_step conditions are highlighted because they are deterministic — they fire before the LLM reads the routing instructions. If any match, the conversation is already on its way.
Routing instructions — LLM
Everything unhighlighted is the LLM's routing logic. Plain language instructions that tell the LLM which action to select for each type of request.
No before or after reasoning blocks
The agent router does not need to stamp a workflow step or clean up after itself. Its only job is routing — it runs, it picks a destination, it transitions. Nothing before, nothing after.
The most important topic in the agent
Every single user message flows through here first. Without it, the agent cannot resume interrupted conversations and has no single entry point. Remove it and the whole agent breaks.
